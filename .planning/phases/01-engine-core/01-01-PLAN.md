```rust
pub enum InboundMobileMsg {
    #[serde(rename = "join")]          Join(MsgJoin),
    #[serde(rename = "pose_frame")]    PoseFrame(MsgPoseFrame),
    #[serde(rename = "calibration_done")] CalibrationDone(MsgCalibrationDone),
    #[serde(rename = "ping")]          Ping(MsgPing),
    #[serde(rename = "pong")]          Pong(MsgPong),
}
```

**Critical notes:**
- `MsgRoundEnd.winner` is `Option<u8>` because TypeScript has `winner: 1 | 2 | null` — use `#[serde(skip_serializing_if = "Option::is_none")]` to omit when None OR serialize as null; check shared/protocol.ts — if it's nullable (not optional), always serialize null: `#[serde(serialize_with = "serialize_nullable_u8")]` or just use `Option<u8>` without skip (serde serializes None as null by default).
- `MsgGameState.hp` and `MsgRoundEnd.final_hp` are tuple `(u32, u32)` — serde_json serializes tuples as `[a, b]` arrays, which matches TypeScript's `[number, number]`. Correct.
- `MsgGameState.wins` is `(u32, u32)` — same tuple pattern; serializes as `[w1, w2]`. Required per FIX-02: win counter must be in the snapshot so the overlay does not lose wins state on reconnect (REQUIREMENTS.md FIX-02, ROADMAP.md success criterion 3). Populate from `state.wins[0]` and `state.wins[1]` in `build_game_state_with_latency`.
- `MsgGameState.poses` is `(Vec<PoseKeypoint>, Vec<PoseKeypoint>)` — tuple of two vecs; serializes as `[[...], [...]]`. Always pass empty vecs in Phase 1 (per `_EMPTY_POSES` pattern from Python).
- Add `use serde::{Deserialize, Serialize};` and `use ts_rs::TS;` at the top.
- Do NOT add `#[ts(export)]` to `InboundMobileMsg`.
- `PoseKeypoint`, `Position`, `HitEvent` need `#[ts(export)]` even though they have no type field — they are referenced by other exported types.

| Boundary | Description |
|----------|-------------|
| Wire protocol | External TypeScript clients send JSON messages over WebSocket; field names/shapes are attacker-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | protocol.rs InboundMobileMsg | mitigate | serde tag discriminator rejects unknown `type` values with a deserialize error; log-and-continue in handler prevents parse error from crashing connection |
| T-01-02 | Information Disclosure | ts-rs export path | mitigate | TS_RS_EXPORT_DIR set in engine/.cargo/config.toml; running `cargo test` overwrites shared/ — reviewers must verify generated output matches protocol.ts intent after each model change |
| T-01-03 | Denial of Service | WebSocket accept | accept | Phase 1 has no auth or rate limiting (per REQUIREMENTS.md out-of-scope); acceptable for local/tunnel deployment model |
| T-01-04 | Spoofing | player_slot in MsgJoin | accept | 6-char room code space (36^6 ≈ 2.2B) is the auth boundary per project security model; no additional slot auth in Phase 1 |
</threat_model>
