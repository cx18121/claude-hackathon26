Read the existing Dockerfile completely first. Then modify it as follows:

1. **Preserve** the existing `overlay-builder` and `mobile-builder` stages exactly as-is.

2. **Insert** a new `engine-builder` stage after the mobile-builder stage:
```dockerfile
# ---- Rust engine build stage ----
FROM rust:1.86-slim AS engine-builder
# Install build dependencies for linking
RUN apt-get update && apt-get install -y --no-install-recommends pkg-config libssl-dev && rm -rf /var/lib/apt/lists/*
WORKDIR /engine
# Copy workspace Cargo files first for layer caching
COPY engine/Cargo.toml ./Cargo.toml
COPY engine/engine-core/Cargo.toml ./engine-core/Cargo.toml
# Create dummy main.rs so cargo can download and cache dependencies
RUN mkdir -p engine-core/src && echo 'fn main() {}' > engine-core/src/main.rs
RUN cargo build --release --manifest-path engine-core/Cargo.toml || true
# Now copy real source
COPY engine/ ./
# Touch main.rs to force rebuild after dependency cache
RUN touch engine-core/src/main.rs && cargo build --release --manifest-path engine-core/Cargo.toml
```

Note: The `rust:1.86-slim` image is Debian-based (bookworm). The final image uses `debian:bookworm-slim` — same glibc version, no exec format errors (Pitfall 5 avoided). Do NOT use `rust:alpine`.

3. **Replace** the final stage. Remove the `FROM python:3.11-slim` stage and everything after it. Replace with:
```dockerfile
# ---- Final image ----
FROM debian:bookworm-slim
RUN apt-get update && apt-get install -y --no-install-recommends ca-certificates && rm -rf /var/lib/apt/lists/*
WORKDIR /app
# Copy Rust binary
COPY --from=engine-builder /engine/target/release/engine-core ./engine-core
# Copy static assets (same paths as Python server)
COPY --from=overlay-builder /app/overlay/dist ./overlay/dist
COPY --from=mobile-builder /app/mobile/dist ./mobile/dist
EXPOSE 8000
CMD ["./engine-core"]
```

Note: The overlay-builder and mobile-builder use `/app/overlay/dist` and `/app/mobile/dist` as output paths — verify these against the existing Dockerfile's COPY targets. Adjust if the existing stages write to different paths.

Note on railway.toml: the `build.builder = "DOCKERFILE"` entry remains unchanged; the Dockerfile switch is transparent to Railway.

Note: Python server files (server/, requirements.txt) are NOT copied into the final image — D-02 says Python server stays at server/ for development but the Dockerfile switches to Rust binary.
