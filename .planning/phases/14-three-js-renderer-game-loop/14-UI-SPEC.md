# Phase 14 — UI Design Contract

> Visual and interaction contract for Phase 14: Three.js Renderer + Game Loop.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none — text-only HUD |
| Font | system-ui, sans-serif (body/labels); "Black Han Sans" or "Anton" from Google Fonts (display/match-end only — bold condensed impact style) |

**Design system note:** No shadcn/radix. This is a game UI. All styling is hand-authored CSS on HTML overlays positioned above the Three.js canvas. Existing `app.css` tokens (`--bg-deep`, `--bg-surface`, `--accent`, `--text-primary`, etc.) are the project's token foundation. Phase 14 extends these with game-specific semantic tokens defined below.

**Font decision:** For all HUD labels (HP values, timer digits, win counters) use `system-ui, sans-serif` at weight 700. For the match-end WIN/LOSE display only, use a bold condensed display font: `"Anton"` from Google Fonts is free, single-weight (400 maps to heavy), and requires no build tooling — load via `<link>` in `index.html`. Source: CONTEXT.md D-05 (Street Fighter / Guilty Gear aesthetic).

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | HP bar inner padding, label gaps |
| sm | 8px | HUD element internal padding |
| md | 16px | HP bar section horizontal margin from viewport edge |
| lg | 24px | HUD top offset from viewport top edge |
| xl | 32px | Match-end overlay padding |
| 2xl | 48px | WIN/LOSE text bottom margin before rematch button |
| 3xl | 64px | Not used in Phase 14 |

**Exception — HP bar height:** 20px (non-standard). This is a game-specific visual element where readability at a glance matters more than strict 4-point alignment. Height is fixed, not part of the spacing scale.

**Exception — canvas fill:** The Three.js canvas is `width: 100vw; height: 100vh`. All HUD elements are `position: absolute` on top of it. No container padding applies to the canvas itself.

## Typography

Four type sizes, two weights. All HUD text is white (`--text-primary`) except where noted.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 700 | 1.2 | Timer digits, HP numeric label (if shown), win counter count |
| Label | 16px | 700 | 1.2 | HP bar player/opponent label ("YOU" / "OPP") |
| Heading | 24px | 700 | 1.1 | Round timer display (top center between HP bars) |
| Display | 80px | 400 (Anton) | 1.0 | WIN / LOSE text on match-end overlay |

**Font stack:**
- HUD labels + timer: `system-ui, -apple-system, sans-serif` at `font-weight: 700`
- WIN/LOSE display: `'Anton', system-ui, sans-serif` at `font-weight: 400` (Anton is inherently heavy)

**Letter spacing:**
- Timer digits: `letter-spacing: 0.04em` (wide for readability)
- WIN/LOSE: `letter-spacing: 0.08em` (bold and airy, fighting game feel)
- All other labels: `letter-spacing: 0.02em`

**Text transform:** All HUD labels UPPERCASE. "YOU", "OPP", "ROUND 1", "WIN", "LOSE", "REMATCH".

## Component Inventory

### 1. HUD Container (`#game-hud`)

- `position: absolute; top: 0; left: 0; width: 100%; height: 100%`
- `pointer-events: none` (except on match-end overlay buttons)
- `z-index: 10` above canvas (`z-index: 1`)

---

### 2. HP Bar — Player Left (`#hp-bar-p1`)

**Layout:**
- `position: absolute; top: 24px; left: 16px`
- Width: `clamp(160px, 28vw, 280px)` — responsive but capped
- Height: `20px`
- Direction: fills left-to-right (full = full bar width)

**Structure:**
```
[YOU]  [████████████░░░░░░░] 
         ^HP fill               ^track (bg)
```

- Label "YOU" above the bar, `font-size: 16px`, `font-weight: 700`, `letter-spacing: 0.02em`, `color: var(--text-primary)`
- Track: `background: var(--hp-bar-bg)`; `border-radius: 4px`
- Fill: `background: var(--hp-full)` when HP > 50%; transitions to `var(--hp-low)` at ≤ 50%
- Fill width: CSS custom property `--hp-pct` driven by React state, `width: calc(var(--hp-pct, 100) * 1%)`. Transition: `width 200ms ease-out, background-color 300ms ease`
- HP starts at 800 (from `engine/boxing-core/src/damage.rs`). `--hp-pct = (currentHP / 800) * 100`

---

### 3. HP Bar — Opponent Right (`#hp-bar-p2`)

Mirror image of P1 bar.

- `position: absolute; top: 24px; right: 16px`
- Width: same responsive formula as P1
- Direction: fills right-to-left (full = full bar width, shrinks from right)
- Label "OPP" above bar, right-aligned
- Same color transition behavior as P1 (green → red at ≤ 50%)
- Opponent HP sourced from `MsgFpsState.hp[opponentIndex]`

---

### 4. Round Timer (`#round-timer`)

- `position: absolute; top: 24px; left: 50%; transform: translateX(-50%)`
- Display: countdown seconds as integer, e.g. `"99"`, `"45"`, `"3"`
- `font-size: 24px; font-weight: 700; letter-spacing: 0.04em; font-family: system-ui, sans-serif`
- `color: var(--text-primary)`
- Background pill: `background: var(--hud-bg); padding: 4px 16px; border-radius: 8px`
- Below the timer, in smaller text (14px weight 700): round win counter `"● ○"` (P1 wins left, P2 wins right). Filled dot = win. Source: GML-01

---

### 5. Hit Flash Overlay (`#hit-flash`)

- `position: absolute; top: 0; left: 0; width: 100%; height: 100%`
- `background: var(--hit-flash)` = `rgba(255,255,255,0.35)`
- `pointer-events: none`
- Default: `opacity: 0; visibility: hidden`
- On `MsgFpsHit` received: `visibility: visible`, animate `opacity: 0 → 0.8 → 0` over `120ms` using CSS `@keyframes` or JS-driven inline style
- The flash is a brief white blip — not a hold. Duration: 120ms total (`30ms rise, 90ms fall`)

---

### 6. Match End Overlay (`#match-end-overlay`)

- `position: absolute; inset: 0`
- `display: flex; flex-direction: column; align-items: center; justify-content: center`
- `background: var(--overlay-dim)` = `rgba(0,0,0,0.65)`
- `pointer-events: auto` (this element accepts clicks — re-enable from HUD container rule)
- Hidden (`display: none`) until match ends; shown when HP reaches zero

**WIN / LOSE text:**
- `font-family: 'Anton', system-ui, sans-serif`
- `font-size: 80px; font-weight: 400; letter-spacing: 0.08em`
- WIN: `color: var(--win-color)` = `#fbbf24`
- LOSE: `color: var(--lose-color)` = `#ef4444`
- Text is always uppercase

**Rematch button:**
- Copy: "REMATCH"
- `margin-top: 48px`
- `padding: 16px 48px`
- `font-size: 16px; font-weight: 700; letter-spacing: 0.04em`
- `background: var(--accent-bright); color: var(--text-primary)`
- `border: none; border-radius: 4px; cursor: pointer`
- Hover state: `filter: brightness(1.15)` — no separate color token needed
- On click: triggers re-calibration flow (sends player back to calibration screen — calls `onRematch()` prop which routes App.tsx back to calibration)
- No confirmation dialog needed — rematch is not destructive

### 7. Three.js Canvas

- Mounted into `<div id="game-canvas-root" />` inside `app-root`

- `z-index: 1`; HUD sits at `z-index: 10` above it

## Interaction Contracts

### HP Bar Width Transition

Trigger: `MsgFpsState` received with updated `hp` values.
Behavior: React state update → CSS `--hp-pct` custom property update on the fill element → `width: 200ms ease-out` CSS transition animates the bar drain.
Color transition: CSS `background-color: 300ms ease` animates green → red when HP drops through 50%.
No JS animation library needed.

### Hit Flash

Trigger: `MsgFpsHit` message received (player's own punch landed, HFB-04).
Behavior: JS sets `element.style.opacity = '0.8'` immediately, then transitions to 0 via a 120ms CSS keyframe animation `hit-flash-anim`.
Implementation: add/remove a CSS class `flash-active` that triggers the keyframe.

```css
@keyframes hit-flash-anim {
  0%   { opacity: 0.8; }
  100% { opacity: 0; }
}
.hit-flash-active {
  animation: hit-flash-anim 120ms ease-out forwards;
}
```

### Camera Shake

Trigger: `MsgFpsHit` received (player took a hit, HFB-01).
Behavior: Three.js only — no CSS. `worldCamera.position` offset driven by Eiserloh trauma-decay pattern (RESEARCH.md Q6). Arms camera is unaffected.

### Opponent Arm Snap Back

Trigger: `MsgFpsHit` received (player landed a punch, HFB-03).
Behavior: Three.js only — target positions for opponent arm joints are forcibly moved to retracted position; high-lambda lerp (50–100) animates the snap. No CSS.

### Match End Display

Trigger: either player's HP reaches 0 (signaled by `MsgFpsState` or match-end event from server).
Behavior: `#match-end-overlay` transitions from `display: none` to `display: flex`. WIN or LOSE text rendered conditionally based on whether `playerSlot` matches the winner.

### Rematch

Trigger: user clicks REMATCH button.
Behavior: `#match-end-overlay` hides; Three.js scene is torn down or paused; App.tsx routes back to CalibrationScreen. No confirmation dialog.
