## Decisions Made

- Placeholder ONNX generated with a real executable graph (Reshape + Gemm with zero weights) rather than empty bytes — passes `onnx.checker.check_model` and has the correct `input`/`logits` names that `usePunchClassifier.ts` expects
- `make_placeholder_onnx.py` included as a committed script so the placeholder can be regenerated deterministically if needed

## Issues Encountered

## Known Stubs

- `fps/public/models/punch_classifier_int8.onnx` is a placeholder with all-zero weights. Every inference will produce uniform logits; confidence will be below 0.7; `usePunchClassifier` will always return `type: null`. This is intentional and documented. The real trained model replaces it after the ml/ pipeline runs on collected training data.
